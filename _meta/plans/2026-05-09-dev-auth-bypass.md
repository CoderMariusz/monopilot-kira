## Caveats

- Next.js loads `.env.local` at startup; restart the dev server after changing `DEV_AUTH_BYPASS`.
- Live curl verification is blocked in this environment by `listen EPERM` on port 3000.
- Full `web` typecheck is blocked by an unrelated React type mismatch in `apps/web/app/[locale]/layout.tsx`.

## User Note

To enable: add `DEV_AUTH_BYPASS=true` to `.env.local`, restart dev server. Production is hardcoded-safe.

## Post-codex follow-up (2026-05-09 by orchestrator)

Codex's middleware patch + tests were correct in isolation, but the bypass did not
take effect in the live dev server. Root cause: **Next 16 + Turbopack evaluates
`next.config.mjs` BEFORE `.env.local` is loaded for the Edge-runtime middleware
bundle**, so `process.env.DEV_AUTH_BYPASS` was `undefined` at config-evaluation
time and the `env:` block in `next.config.mjs` inlined an empty string into the
middleware bundle.

Two follow-up fixes were applied to `apps/web/next.config.mjs`:

1. Added a manual `.env.local` parser at the top of the config file. It checks
   both `apps/web/.env.local` (per-app) and the workspace root `.env.local`
   (monorepo). Shell-set vars are not overridden.
2. Added an `env:` block exporting `DEV_AUTH_BYPASS` so the value is inlined
   into the Edge-runtime middleware bundle.

Verification after follow-up (no shell env, only `.env.local`):

```text
$ curl -sS -o /dev/null -w "%{http_code}\n" -H "Authorization: Bearer fake" \
       http://localhost:3000/en/settings
```
