## Breaking changes — migration notes

### Next.js 15 → 16

- `pnpm create next-app@16` (all scaffold tasks: T-001 in 00-foundation)
- `import { cacheLife, cacheTag } from 'next/cache'` (remove `unstable_` prefix)
- React 19.2 included — View Transitions API available natively
- Upgrade command: `pnpm next upgrade`

**Tasks to update:** `00-foundation/T-001`, `00-foundation/T-041` (PWA), any task with `next.config.mjs`

### Tailwind CSS v3 → v4

- **No more `tailwind.config.ts`** — config lives in CSS:
  ```css
  @import "tailwindcss";
  @theme { --color-brand: #1e40af; }
  ```
- Variant stacking order reversed: `first:*:pt-0` → `*:first:pt-0`
- `hover:` applies only on hover-capable devices (use `@custom-variant hover (&:hover)` to restore v3 behavior)
- Unified toolchain — no separate PostCSS config needed
- Up to 5x faster full builds, 100x faster incremental

**Tasks to update:** `00-foundation/T-004` (Tailwind init), `00-foundation/T-030`/`T-031` (UI primitives), all T3-ui tasks referencing `tailwind.config.ts`

### Zod v3 → v4

- `z.string().min(5, { message: "Too short" })` → `{ error: "Too short" }`
- `{ invalid_type_error: "...", required_error: "..." }` → `{ error: (issue) => ... }`
- Both v3 and v4 can coexist: `import * as z3 from "zod/v3"` (for gradual migration)

**Tasks to update:** `00-foundation/T-017` (json-schema-to-zod runtime), `00-foundation/T-018` (Rule DSL), all T2-api and T3-ui tasks with Zod resolvers

### Vitest v3 → v4

- Browser context: `import { page } from '@vitest/browser/context'` → `import { page } from 'vitest/browser'`
- `--standalone` with filename filter now auto-runs matched files
- Upgrade: `pnpm add -D vitest@4`

**Tasks to update:** `00-foundation/T-013` (Vitest config), all T4-wiring-test tasks with browser tests
