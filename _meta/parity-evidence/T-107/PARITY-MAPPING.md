## Deviations from prototype (intentional)

1. **Mock data → real Supabase data.** Prototype's `GATE_CHECKLISTS` constant + `overrides` local state
   replaced by real `gate_checklist_items` read server-side via the T-057 `getProject` Server Action
   (org-scoped, RLS `app.current_org_id()`); the RSC parent (T-111) maps `checklistByGate` → the `gates`
   prop. The client island never queries the DB (risk red-line honored).
2. **`<input type=checkbox>` → `@monopilot/ui Checkbox`** (`role=checkbox`, keyboard-operable) per shadcn red-line.
3. **a11y — status never color-only.** Blocking/Current/Required badges and the blocker/ready alerts all
   pair color with a glyph (⚠ / ✓) + text label. Done items get `line-through` AND an accessible meta line.
4. **Progress bars get `role=progressbar` + aria values** (prototype used bare divs).
5. **Toggle is optimistic** with rollback on Server Action failure (`toggleGateChecklistItem`, owned by T-058,
   passed in as a prop — never authored here). Disabled while `canWrite=false` (RBAC server-resolved).
6. **Gate sequence/metadata** (GATE_ORDER, next/nextLabel/requiresApproval) is a domain constant supplied per
   gate via the `gates` prop (parent derives it from `npd_projects.current_gate` + the G0-G4 domain model);
   not hardcoded inside the island.
7. **i18n** — all visible strings come from `npd.gateChecklist` (en/pl/ro/uk), no inline literals.

## Playwright / axe

Route-level Playwright parity + axe is owned by **T-112** (explicitly out of scope for T-107) and the
route page by **T-111**. Per `_meta/atomic-tasks/UI-PROTOTYPE-PARITY-POLICY.md` the documented fallback
(RTL parity checklist + per-state DOM snapshots) is supplied here. a11y is enforced at component level:
`role=progressbar`/`role=alert`/`role=status`, `aria-expanded`+`aria-controls` on collapsibles, `role=checkbox`
with `aria-label`+`aria-describedby`, and color-never-alone status signalling (asserted by RTL).
