# Decision — MonoPilot localized admin routing

Date: 2026-05-20
Status: accepted
Scope: `apps/web` user-visible admin/account routes, starting with `02-settings`.

## Decision

MonoPilot `apps/web` uses `next-intl` locale routing through `apps/web/proxy.ts`. Therefore every user-visible browser route must be implemented under `apps/web/app/[locale]/...`.

For admin/settings/account pages, the canonical source shape is:

- `apps/web/app/[locale]/(admin)/settings/.../page.tsx`
- `apps/web/app/[locale]/(admin)/account/.../page.tsx`

The browser URL shape remains route-group-free:

- `/{locale}/settings/...`
- `/{locale}/account/...`

Example:

- canonical source: `apps/web/app/[locale]/(admin)/settings/users/page.tsx`
- canonical URL: `/en/settings/users`

## Rejected alternative

Do not implement user-visible Settings/account pages only under:

- `apps/web/app/(admin)/settings/...`
- `apps/web/app/(admin)/account/...`

Those paths can make co-located RTL tests pass while localized browser URLs such as `/en/settings/users` are missing or drift from the real route.

## Temporary compatibility rule

A legacy non-localized route may remain only as a redirect shim, not as the production UI implementation. Example:

- `apps/web/app/(admin)/settings/users/page.tsx` may redirect to `/en/settings/users`.
- It must not contain the full page UI or be accepted as T3 UI closeout evidence.

## Closeout gate

For any browser-visible Settings/account UI task:

1. `scope_files` and test commands must point to `[locale]/(admin)` source paths.
2. The localized page source must exist at the canonical path.
3. Legacy `(admin)` page files must be absent or explicit redirect shims.
4. Links generated by the UI must preserve the active locale or use a locale-aware route helper.
5. Closeout evidence must use the localized URL (`/{locale}/settings/...` or `/{locale}/account/...`) and include the UI parity artifacts required by `_meta/atomic-tasks/UI-PROTOTYPE-PARITY-POLICY.md` for prototype-backed tasks.

## Initial application

Applied first to `02-settings` task metadata and T-059 Users screen remediation.
