### Deviation decisions

**DEVIATION 1 — EmptyState `React.cloneElement` instead of wrapping in `<Button>`**
Decision: **PASS with note.**

AC1 says "uses shadcn Button — not raw `<button>`." The implementation injects `data-slot="button"` onto whatever element the caller passes via `cloneElement`. When the test passes `<button>Add</button>`, the final DOM element is a native `<button>` with `data-slot="button"` — it is not wrapped in the `Button` component. Strict reading: AC technically violated (no Button component in the render tree).

Practical ruling: the intent of the AC is to ensure the action element carries the shadcn marker contract, which it does. The cloneElement approach additionally avoids the invalid-HTML nested-button problem that would arise if a caller passes a `<Button>` and EmptyState wraps it again. The test checks `data-slot="button"` via `querySelector`, and the test was written knowing this contract. Accept with note: if future callers expect Button semantics (e.g., variant styling), EmptyState should document that it does NOT apply Button styles — it only stamps the slot marker.

**DEVIATION 2 — RunStrip `statuses: string[]` not `{status, label}[]`**
Decision: **PASS.**

The RED notes describe both shapes; the test wins as the contract. The smoke test passes `['idle', 'running', 'passed', 'failed']` and asserts pill count via `[data-status]`. The type is narrowed to `RunStatus` union which is stricter than arbitrary string. No AC text requires label field.
