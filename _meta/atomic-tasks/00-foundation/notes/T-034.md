Notes for implementer:

1. **Module surface (`packages/ops/src/drift-detect.ts`)**
   - Export `async function detectDrift(opts: { orgId: string; deptCode?: string; tableMap?: Record<string,string> })`.
   - Return shape: `{ diff: { missing_in_db: string[]; extra_in_db: string[]; type_mismatch: Array<{ column: string; expected: string; actual: string }> } }`.
   - Implementation steps per JSON spec:
     a. Load all `Reference.DeptColumns` rows (filter by orgId; deptCode optional for full sweep).
     b. Resolve dept_code → physical table; for the foundation slice, accept an injected `tableMap` (e.g. `{ core: 'public.organizations' }`) so RED tests can avoid creating extra DDL. Production may default to a built-in mapping or read a separate registry.
     c. Query `information_schema.columns` for the resolved table(s).
     d. Compute three-way diff:
        - `missing_in_db`: column_keys present in DeptColumns but absent from information_schema.
        - `extra_in_db`: columns present in information_schema but absent from DeptColumns (only when a strict mode flag is on — out of scope for AC1).
        - `type_mismatch`: column_keys present in both but FieldTypes.ts_type ≠ information_schema.data_type (after canonical mapping).
     e. If `diff.missing_in_db.length + diff.type_mismatch.length > 0`, INSERT one `audit_events` row:
        - `action='schema.drift_detected'`
        - `resource_type='reference.dept_columns'`, `resource_id=<dept_code or '*'>`
        - `retention_class='operational'` (CHECK constraint enforces this — see SQLSTATE 23514 mutation test)
        - `after_state=jsonb_build_object('diff', diff, 'org_id', orgId)`
        - `request_id=<uuid generated for the run>`
     f. Return `{ diff }` regardless.

2. **Connection**
   - Use `getOwnerConnection()` from `@monopilot/db` test-utils OR construct a pool via `DATABASE_URL`. The job runs as the system actor; per T-058 prefer `getOwnerConnection` for DDL-equivalent reads (information_schema) and `getAppConnection` only if RLS scoping is desired. T-034 audits as system → owner pool is acceptable. Add `@monopilot/db` to `packages/ops/package.json` deps when implementing.

3. **Cron route (`apps/web/app/api/internal/cron/drift/route.ts`)**
   - Export `GET(req: Request): Promise<Response>`.
   - Auth: 401 unless EITHER `x-vercel-cron: 1` is present OR `Authorization: Bearer ${process.env.CRON_SECRET}` matches. The mutation test sends both — implementer can pick which signal is canonical (recommendation: require Bearer in production, allow `x-vercel-cron` in preview).
   - On auth pass: call `detectDrift({ orgId: <system-org or iterate all orgs> })`, return JSON `{ ok: true, diff }`.
   - DO NOT couple to Next.js types in the test — the test uses Web `Request`/`Response` directly (Node 20 has them globally).

- **AC1 — drift → audit row**: PASS. `detectDriftImpl` aggregates `missing_in_db` + emits a single `audit_events` INSERT with `action='schema.drift_detected'`, `retention_class='operational'`, `actor_type='system'`, `after_state` jsonb. ### Deviation evaluations
- **A — 2 narrow ESLint disables**: ACCEPT. Rationale is sound (RLS-scoped pool incompatible with system-actor cross-org sweep; getOwnerConnection barred by T-045 outside migrations). Inline comments at drift-detect.ts:172–175 and route.ts:83–87 link the T-058 / T-009 reasoning. **Carry-forward T-080**: `getSystemActorConnection()` helper in `@monopilot/db` so future system-pool callers cannot copy the disable pattern.
- **B — `extra_in_db` empty**: ACCEPT. JSON `out_of_scope` does not require strict mode; AC1 only pins `missing_in_db = ['foo']`. Gap is documented at drift-detect.ts:136–139. **Carry-forward T-081**: enable strict mode + `extra_in_db` population once table-map registry exists.
- **C — relative cron import**: ACCEPT. No alias is wired; relative path is correct + lint-clean. **Carry-forward T-082**: add `@monopilot/ops` alias to `apps/web/tsconfig.json` and swap to named import.

### Security findings
- **CRON_SECRET fail-closed**: VERIFIED at route.ts:35–57 — production with unset secret rejects all Bearer.
- **`x-vercel-cron` spoofability**: NOT addressed. On Vercel the platform strips the header from external requests, but on self-hosted Next.js it is freely forgeable. **Action: nit** — add a comment pinning Vercel-only deploy, or layer Bearer-required-in-prod even when the header is present. Non-blocking for AC3 since the JSON only requires "internal cron header".
- **Bearer comparison non-constant-time** (route.ts:48 uses `===`). Leaks ~ns-scale timing. **Action: nit** — switch to `crypto.timingSafeEqual` over equal-length Buffers. Non-blocking but should be tracked under T-080 alongside the helper extraction.
