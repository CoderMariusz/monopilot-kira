Deviations from PRD / notes:
1. **@serwist/next install** — confirmed blocker; resolved with `pnpm --filter web add @serwist/next serwist`. Both added as runtime deps (serwist is used in sw.ts at SW runtime; @serwist/next needed for next.config.mjs build plugin).
2. **sw.ts node-env guard** — Vitest runs in `environment: 'node'`; `new Serwist({ navigationPreload: true })` calls `self` synchronously in the constructor. Guard wraps entire instantiation in `typeof self !== 'undefined' && 'ServiceWorkerGlobalScope' in globalThis`. This is test-only safety; real builds compile sw.ts in a SW context where `self` exists.
3. **Plugin wrap order** — `withNextIntl(withSerwist(nextConfig))`. Serwist inner (adds webpack InjectManifest plugin), next-intl outer (adds i18n routing middleware). Either order is functionally equivalent for separate concerns; this order mirrors the @serwist/next docs convention.
4. **sw.ts navigationPreload omitted** — removed from Serwist constructor options to avoid the `self` reference guard complexity in node env; can be re-added in T-042 E2E context.
