## Phase: REVIEW
Status: **REWORK**

Verdict: **REWORK** — one red-line violation and one structural defect.

### Findings

**[FAIL — RED LINE] VisuallyHidden uses inline styles**
`Summary.tsx` lines 22-35: the `VisuallyHidden` helper renders via `style={{ clip, clipPath, height, overflow, position, whiteSpace, width }}` — seven inline-style properties. The PRD risk red line states: "Do not use inline styles for diff colours — must come from tokens.css for tenant theming hook." The AC1 requirement broadens this: "uses tokens.css for spacing — no inline styles." The no-inline-style tests only check the root element (`container.firstElementChild`) and `[data-summary-row]` wrappers; neither selector reaches the `<span>` inside `<dt>`. The visually-hidden span therefore carries inline styles undetected by the test suite. The `.summary-status-label` class is defined in `Summary.module.css` exactly for this purpose but is never applied to the span. Fix: replace the inline `style` object with `className="summary-status-label"` (after confirming `generateScopedName: '[local]'` produces the correct class name).

**[FAIL — test gap / vacuous scan] "No inline styles" assertion does not cover visually-hidden span**
The test at line 95-104 asserts `root.style.length === 0` (the `<dl>`) and lines 106-121 assert `[data-summary-row]` wrappers. Neither reaches the `<span>` inside `<dt>`. A mutation that adds inline styles to that span passes both tests. The test should also assert `container.querySelectorAll('[style]').length === 0` across the full subtree, or at minimum enumerate `dt > span` elements.

**[PASS with note] `vitest.config.ts` `generateScopedName: '[local]'`**
Setting is correct for removing CSS module hash suffixes so plain class-name string assertions work. No evidence this breaks existing Modal tests (GREEN notes confirm 18/18 Modal tests continue to pass).

**[PASS] AC1 structural parity**
`<dl>`/`<dt>`/`<dd>` rendered; order tested with both forward and reverse mutation; `after` value in `<dd>` verified; no interactive controls; tag-name assertions correct.

**[PASS] AC1 order mutation-proof**
Separate test re-renders with `[...rows].reverse()` and asserts reversed `<dt>` text. Mutation-proof.

**[PASS] AC2 token assertion**
File-content assertions on `tokens.css` for `--color-warning` and on `Summary.module.css` for `var(--color-warning)` are present; `tokens.css` now defines `--color-warning: #d97706`. CSS module correctly uses `var(--color-warning)` for `.summary-row--changed`.

**[PASS] AC2 a11y — "changed" in accessible name**
`VisuallyHidden` renders ` (changed)` text inside `<dt>`; `screen.queryByText(/changed/i)` finds it. Mechanism works even though the implementation is via inline styles (the a11y assertion itself passes; the inline-style violation is a separate defect).

**[PASS] AC3 empty rows**
Renders without throwing; `[role="status"]` / "No changes" present; no `<dt>`/`<dd>`; custom `emptyState` prop accepted.

**[NOTE] Summary.module.css spacing — tokens used for border width only**
The CSS module defines only `border-left` rules (no padding/margin). There are no spacing rules to verify against `--space-*` tokens; the `<dl>`/`<dt>`/`<dd>` default browser spacing is used. AC1 says "uses tokens.css for spacing" — this is marginal but likely acceptable given the component is read-only structural primitive. Not blocking.

**[NOTE] Stories EmptyCustom uses inline style**
`Summary.stories.tsx` line 91 renders `<p style={{ color: '#6b7280' }}>` as the custom `emptyState` value. This is caller-supplied content passed via prop, not part of Summary itself — not a violation of the red line.

### Required fixes before CLOSEOUT

2. Extend the no-inline-styles test to assert `container.querySelectorAll('[style]').length === 0` (or equivalent full-subtree check), so the visually-hidden span is covered.
