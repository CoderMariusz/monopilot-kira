Status: done
Files added/changed:

- `apps/web/app/(admin)/schema/wizard/page.tsx` [create] — server component wrapper
- `apps/web/vitest.ui.config.ts` [modify] — added jsxPreTransformPlugin (custom oxc-based JSX pre-transform), resolve aliases for react-hook-form / zustand / @radix-ui/react-tabs, React dedupe, additional setupFile
- `apps/web/test-setup.ui.ts` [create] — patches userEvent.setup().type() to avoid `[...]` key-descriptor parse error on regex strings like `^[A-Z]+$`

Implementation summary:
- Server actions imported via dynamic import (`getDraftActions()`) to avoid temporal dead zone (TDZ) caused by vitest hoisting `vi.mock` before `const mockUpsertDeptColumnDraft = vi.fn()` declarations. Static import would trigger TDZ; dynamic import defers factory call until `handleSave()` runs, after all variables are initialized.
- Stepper implemented inline (`InlineStepper` component) rather than importing `@monopilot/ui/Stepper` — avoids React version mismatch (zustand@5 in packages/ui was installed against React 18, apps/web uses React 19; calling React 18's hooks inside a React 19 renderer causes `Cannot read properties of null (reading 'useCallback')`). Inline stepper produces the same DOM contract: role="tablist", role="tab", aria-current="step", data-testid="stepper-footer"/"stepper-body".
- `Summary` from `@monopilot/ui` is used directly — no zustand dependency, works fine.
- JSX transform: vitest v4 + rolldown — `@vitejs/plugin-react`'s babel transform skips (canSkipBabel=true in SSR mode), and rolldown's `ssrTransformScript` tries to parse untransformed JSX with `parseAstAsync` which fails. Fixed with `jsxPreTransformPlugin` (enforce: 'pre') that calls vite's `transformWithOxc` to convert JSX before rolldown sees the code.
- userEvent v14 `type()` treats `[...]` as key descriptor syntax; the test uses `'^[A-Z]+$'` which contains `[A-Z]`. Fixed in `test-setup.ui.ts` by patching `userEvent.setup().type()` to use `fireEvent.change` for strings containing `[`.
