Notes for implementer:
- **Stepper composes inside Modal — does NOT replace it (red line).** Every wizard usage wraps Stepper with a Modal; Stepper is a layout primitive inside the modal body.
- **New deps required** — run before implementing:
  ```
  pnpm --filter @monopilot/ui add @radix-ui/react-tabs zustand
  ```
  Neither package exists in `packages/ui/package.json` yet. `zustand` is NOT installed anywhere in the monorepo.
- **Prop surface** (derived from tests — implement exactly this shape):
  ```ts
  export interface StepDef {
    id: string;
    label: string;
    canEnter?: () => boolean;   // absent = implicitly enterable
  }

  interface StepperProps {
    wizardId: string;            // key for stepperStore
    steps: StepDef[];
    currentStep: number;         // index, 0-based
    onChange: (nextStep: number) => void;
    hasErrors?: boolean;         // disables Next when true
    cancelSlot?: React.ReactNode; // render-prop pattern for Cancel button
    onEscape?: () => void;       // called on Escape keydown (Modal contract)
    children?: React.ReactNode;  // body slot
  }
  ```
- **stepper-store.ts** must export `useStepperStore` (Zustand store) with:
  - `getStep(wizardId: string): number`
  - `setStep(wizardId: string, step: number): void`
  - `reset?(wizardId: string): void` — needed by tests' beforeEach
  - State shape: `{ steps: Record<string, number> }`
- **Required data-testid attributes** on Stepper output:
  - `data-testid="stepper-root"` — outermost wrapper
  - `data-testid="stepper-body"` — body slot container
  - `data-testid="stepper-footer"` — footer container
- **aria-current="step"** must be set on the active Radix Tabs.Trigger (not aria-selected only).
- **Back button**: `aria-disabled="true"` at step 0; clicking does nothing (do NOT use `disabled` attribute alone — use `aria-disabled` + `onClick` guard as per test assertions).
- **Next button**: same pattern — `aria-disabled="true"` + onClick guard when `hasErrors=true`.
- **Jump (tab click)**: when `canEnter()` returns false, the Tabs.Trigger must have `aria-disabled="true"` and its click must be intercepted (call `e.preventDefault()` in `onSelect` / prevent tab change) so `onChange` is never called.
- **ESC handling**: attach `onKeyDown` to `stepper-root` that calls `onEscape?.()` on Escape. This satisfies the Modal contract — the parent Modal's own ESC handler also fires (Radix Dialog default); Stepper must not swallow the event.
- **Radix Tabs note**: `Tabs.Root value` should be controlled by `currentStep` index cast to string. Override `onValueChange` to call `onChange(index)` only if `canEnter()` returns true for the target step.
- **Modal does NOT use Dialog.Portal** (T-056 carry-forward) — when Stepper is used inside Modal in tests, query via `container.queryBy*` exactly like `Modal.test.tsx` does.
- **tokens.css density**: apply `--space-4` / `--space-2` gap to nav tabs and footer via inline style or CSS module. `data-density` attribute on stepper-root is acceptable evidence for AC4 visual review.
- **Storybook story** (`packages/ui/.storybook/Stepper.stories.tsx`) and Playwright parity are GREEN/CLOSEOUT phase concerns — not RED.

Implementation summary:
- `stepper-store.ts`: Zustand `create` store with `steps: Record<string, number>` shape. `getStep` returns 0 for unknown ids. `reset` deletes the key. Store is a singleton accessible outside React via `.getState()`.
- `Stepper.tsx`: Uses `@radix-ui/react-tabs` (Tabs.Root/List/Trigger) for nav. Controlled by `currentStep` prop (cast to string value). `onValueChange` guards against locked steps (canEnter()=false). aria-disabled on locked tabs + onClick preventDefault. Back/Next use aria-disabled + onClick guards (not native `disabled`). onKeyDown on stepper-root calls onEscape on Escape without swallowing event. useEffect seeds store on each currentStep change.
