Notes for implementer:

### Migration
- File: `packages/db/migrations/019-pins.sql` (NOT `008-pins.sql`)
- Table DDL:
  ```sql
  CREATE TABLE IF NOT EXISTS public.user_pins (
    user_id        uuid        PRIMARY KEY REFERENCES public.users(id) ON DELETE CASCADE,
    pin_hash       text        NOT NULL,
    attempts_count int         NOT NULL DEFAULT 0,
    locked_until   timestamptz,
    last_attempt_at timestamptz
  );
  ALTER TABLE public.user_pins ENABLE ROW LEVEL SECURITY;
  ALTER TABLE public.user_pins FORCE ROW LEVEL SECURITY;
  -- RLS policy: USING (user_id IN (SELECT id FROM public.users WHERE org_id = app.current_org_id()))
  -- GRANT SELECT, INSERT, UPDATE, DELETE ON public.user_pins TO app_user;
  ```
  RLS choice: `user_id IN (SELECT id FROM users WHERE org_id = app.current_org_id())` matches the
  pattern from 018-password-history.sql and 011-departments.sql. The simpler `USING true` would
  not protect cross-org leakage if org context is ever mis-set; the org-scoped subquery is safer.

### Module: `packages/auth/src/verify-pin.ts`
Exports:
```ts
export async function setPin(userId: string, pin: string): Promise<void>
export async function verifyPin(userId: string, pin: string): Promise<true | false | 'locked'>
```

argon2 params:
```ts
import * as argon2 from 'argon2';
const ARGON2_OPTS = {
  type: argon2.argon2id,
  memoryCost: 65536,   // 64 MiB
  timeCost: 3,
  parallelism: 1,
};
```
These match the T-061 pattern. `argon2` is already in `packages/auth/package.json` (verified).

Lockout logic in `verifyPin`:
1. SELECT `pin_hash, attempts_count, locked_until, last_attempt_at` FROM `user_pins` WHERE `user_id=$1`
2. If `locked_until IS NOT NULL AND locked_until > now()`: return `'locked'`
3. Call `argon2.verify(pin_hash, pin)`
4. If verify succeeds: reset `attempts_count=0, locked_until=NULL`; return `true`
5. If verify fails: count failures in window = `attempts_count` (if `last_attempt_at > now()-10min`) else reset to 0; increment; `last_attempt_at = now()`
6. If new `attempts_count >= 6`: set `locked_until = now()+15min`; return `'locked'`
7. Else return `false`

(AC2 test spec: 5 failures → false, 6th → 'locked'. So condition to lock is when the attempt count
**after increment** reaches 6, i.e. `attempts_count >= 6` post-increment, or equivalently when the
failure being recorded is the 6th within the window.)
