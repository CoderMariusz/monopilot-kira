### RBAC integration pattern (from T-014)
The session lookup should resolve the caller's roles from `public.user_roles` joining `public.roles` on `org_id` + `slug = 'org.access.admin'`. See `packages/rbac/src/grant.ts` for the DB query pattern. The `Permission.ORG_ACCESS_ADMIN` constant is `'org.access.admin'`.

### posthog-node API note
`posthog-node` v4+ `isFeatureEnabled` is async (returns `Promise<boolean | undefined>`).
The `PostHog` constructor signature:
```typescript
new PostHog(apiKey: string, options?: PostHogOptions)
```
Use `POSTHOG_API_KEY` and `POSTHOG_HOST` env vars for the self-hosted instance.
Server-side only — do NOT import posthog-node in any client component.
