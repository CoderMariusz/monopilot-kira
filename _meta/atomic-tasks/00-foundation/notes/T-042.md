### AC2 — sw.ts offline navigation
The `NavigationRoute` or `navigationPreload` must be present. Since AC4 requires `navigationPreload: true`, adding that to the Serwist constructor satisfies both AC2 and AC4 (navigationPreload is what enables offline HTML navigation in Serwist via the network-first preload path).

### AC3 — node env blocker for IDB tests
The root `vitest.config.ts` uses `environment: 'node'`. IndexedDB and `window` are unavailable in node. Two options:
1. **Recommended**: Add a `@vitest/browser` or jsdom environment annotation to the test file:
   ```ts
   // @vitest-environment jsdom
   ```
   And install `fake-indexeddb` in `apps/web/devDependencies` (or use the one in `packages/sync-queue`).
2. Move AC3 tests to a separate config that uses `vitest.ui.config.ts` (jsdom).

The test imports from `../../../packages/sync-queue/src/index.js` directly (workspace source). The implementer may need to adjust the import path or add a workspace alias.

### AC4 — sw.ts changes required
1. Add `navigationPreload: true` to the Serwist constructor:
   ```ts
   const serwist = new Serwist({
     precacheEntries,
     skipWaiting: true,
     clientsClaim: true,
     navigationPreload: true,   // <-- ADD
     runtimeCaching: [
       {
         matcher: ({ url }) => url.pathname.startsWith('/api/'),
         handler: new NetworkFirst({ networkTimeoutSeconds: 5 }),  // <-- OVERRIDE defaultCache's 10s
       },
       ...defaultCache,
     ]
   });
   ```
2. Import `NetworkFirst` from `serwist`:
   ```ts
   import { Serwist, NetworkFirst } from 'serwist';
   ```
3. The test asserts `networkTimeoutSeconds: 5` (not 10) AND that `/api/` appears as a custom entry literal in `sw.ts`. Both are required.

### Mutation experiments (test catches)
- **AC1**: Remove `categories`/`screenshots` → assertion fails on `hasScreenshots || hasCategories`
- **AC2**: Remove `NavigationRoute`/`navigationPreload` → assertion fails on `hasNavigationRoute || hasNavigationPreload`
- **AC3**: Don't call `enqueue()` → `after - before === 0` → fails `toBe(1)`. Don't bind `online` event → queue not drained → fails `toBe(0)`
- **AC4**: Use 10s timeout → fails `toMatch(/networkTimeoutSeconds\s*:\s*5/)`. Remove `navigationPreload` → fails `toMatch(/navigationPreload\s*:\s*true/)`. Use 10s and assert `not.toMatch(/networkTimeoutSeconds\s*:\s*10/)` → assertion catches both fails simultaneously.
