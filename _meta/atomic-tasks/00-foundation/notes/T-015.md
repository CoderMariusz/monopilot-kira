### Migration: `packages/db/migrations/007-mfa.sql`

```sql
-- mfa_secrets: one row per enrolled user; secret encrypted with libsodium secretbox
CREATE TABLE IF NOT EXISTS public.mfa_secrets (
  user_id          uuid        PRIMARY KEY REFERENCES public.users(id) ON DELETE CASCADE,
  secret_encrypted text        NOT NULL,  -- base64(libsodium.crypto_secretbox_easy(rawSecret, nonce, key))
  enrolled_at      timestamptz NOT NULL DEFAULT now()
);

-- recovery_codes: 10 codes per user; argon2id hashed; one-time use (used_at set on first use)
CREATE TABLE IF NOT EXISTS public.recovery_codes (
  id         bigserial   PRIMARY KEY,
  user_id    uuid        NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  code_hash  text        NOT NULL,        -- $argon2id$v=19$m=65536,t=3,p=1$...
  used_at    timestamptz,                 -- NULL = unused; non-NULL = used
  created_at timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE public.mfa_secrets     ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.mfa_secrets     FORCE ROW LEVEL SECURITY;
ALTER TABLE public.recovery_codes  ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.recovery_codes  FORCE ROW LEVEL SECURITY;

-- RLS: org-scoped via JOIN to users.org_id (same pattern as 018-password-history.sql)
CREATE POLICY mfa_secrets_org_context ON public.mfa_secrets
  USING (user_id IN (SELECT id FROM public.users WHERE org_id = app.current_org_id()));

CREATE POLICY recovery_codes_org_context ON public.recovery_codes
  USING (user_id IN (SELECT id FROM public.users WHERE org_id = app.current_org_id()));

GRANT SELECT, INSERT, UPDATE ON public.mfa_secrets    TO app_user;
GRANT SELECT, INSERT, UPDATE ON public.recovery_codes TO app_user;
```

await sodium.ready;

  randomBytes(6).toString('base64url').slice(0, 8),

argon2id params (SAME as T-016 / T-061):
```ts
import * as argon2 from 'argon2';
const ARGON2_OPTS = {
  type: argon2.argon2id,
  memoryCost: 65536,  // 64 MiB — mutation m=4096 → AC argon2id tests fail
  timeCost: 3,
  parallelism: 1,
};
```

One-time enforcement in verifyRecoveryCode:
1. `SELECT id, code_hash, used_at FROM recovery_codes WHERE user_id=$1 AND used_at IS NULL`
2. For each row: `argon2.verify(code_hash, code)` — first match wins
3. If match found and `used_at IS NULL`: `UPDATE recovery_codes SET used_at=now() WHERE id=$1`; return `true`
4. If match found but `used_at IS NOT NULL` (already used): insert audit_events row with
   `action='mfa.recovery_replay_attempt'`, `retention_class='security'`; return `false`
5. No match: return `false`

Audit event INSERT for replay attempt:
```sql
INSERT INTO public.audit_events
  (org_id, actor_user_id, actor_type, action, resource_type, resource_id, request_id, retention_class)
VALUES (
  (SELECT org_id FROM public.users WHERE id = $1),
  $1, 'user', 'mfa.recovery_replay_attempt',
  'mfa_recovery_code', $1,
  gen_random_uuid()::text, 'security'
)
```

### Blockers for implementer

1. **libsodium init**: `libsodium-wrappers` requires `await sodium.ready` before any call.
   Must be awaited at module initialization or lazily before each use.
   Alternative: use `libsodium-wrappers-sumo` for synchronous access.

2. **Key derivation for MASTER_KEY**: Tests pass `masterKey` as 64-char hex string (32 bytes).
   Implement HKDF (Node.js `crypto.hkdf`) with `salt=tenantId`, `info='mfa-secret-key'`, length=32.
   This is omitted from test assertions (testing DB storage, not key math) but must be consistent
   across enroll/verify calls.

3. **WebAuthn import strategy**: @simplewebauthn/server is NOT in package.json. Do not add it.
   The stub MUST return the deferred response without any import of WebAuthn libraries.
   The test spies on `globalThis.fetch` to detect any network call — fail fast on any HTTP request.

4. **Test seed dependency**: The tests seed `tenants`, `organizations`, and `users` tables.
   Requires migrations 001–006 to be applied (test DB setup). The `public.audit_events` table
   must exist (migration 004) for AC2 audit row assertions. The `public.mfa_secrets` and
   `public.recovery_codes` tables require migration 007-mfa.sql.

5. **enrollWebAuthn return type**: Must return `{ disabled: true as const, reason: 'phase_3_deferred' as const }`
   — the test checks `Object.keys(result).length === 2` so no extra fields allowed.
