### `packages/ui/src/Field.tsx` — implementation contract
Props: `{ name: string; label: string; hint?: string; required?: boolean; type: string; schema?: z.ZodTypeAny }`

- Internally calls `useFormContext()` (caller wraps in `<FormProvider>`).
- Uses RHF `<Controller>` to register the field.
- Mode must be `'onBlur'` — the FormWrapper helper in tests calls `useForm({ mode: 'onBlur' })`.
- On validation error: set `aria-invalid="true"` on the input element; render the resolver error message below the input (replacing hint text).
- When no error and hint is provided: render hint below the input.
- The inner input must NOT be a raw `<input>` — it must be wrapped by an `Input` component that carries `data-slot="input"` on its root element.
- Required asterisk: `<span aria-label="required" tabIndex={-1}>*</span>` rendered inside or adjacent to the label. Non-interactive (not button/a, no positive tabIndex).
- `aria-describedby` on the input must point to the error element's `id` when an error is present.
- Label must be associated to the input (`htmlFor` / `id` pair) so `getByLabelText` resolves.

### `packages/ui/src/Input.tsx` — hand-roll a minimal shadcn-style wrapper (RECOMMENDED)
Shadcn is not installed. Create a minimal wrapper:
```tsx
// packages/ui/src/Input.tsx
import React from 'react';

export interface InputProps extends React.InputHTMLAttributes<HTMLInputElement> {}

const Input = React.forwardRef<HTMLInputElement, InputProps>((props, ref) => (
  <span data-slot="input" style={{ display: 'contents' }}>
    <input ref={ref} {...props} />
  </span>
));
Input.displayName = 'Input';
export default Input;
```
This satisfies the AC1 assertion that walks ancestors looking for `data-slot="input"`.

### `packages/ui/package.json` exports to add
Add `"./Field": "./src/Field.tsx"` (and optionally `"./Input": "./src/Input.tsx"`) to the exports map.

### Prototype reference
Canonical structure from `prototypes/design/Monopilot Design System/settings/access-screens.jsx:139-145`:
```jsx
<div className="field">
  <label>Email address</label>
  <input type="email" placeholder="name@apex.pl" autoFocus />
</div>
```
Field must match: label-on-top → Input wrapper → hint/error below.

### RHF `useFormContext` dependency
Field uses `useFormContext()` — the caller MUST wrap with `<FormProvider>`. The test helper `FormWrapper` does this automatically. If `useFormContext()` is called outside a provider it throws at render time.
