import { createServerClient } from '@supabase/ssr';
import createIntlMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';
import { checkIdleTimeout } from './lib/auth/session-check';

const intlHandler = createIntlMiddleware(routing);

**DEVIATION A [HIGH] — middleware does NOT call `app.set_org_context` → REWORK-blocking.**

This means every future Server Action author must remember to manually call the SQL function — exactly the foot-gun the risk_red_lines warn against ("Do not bypass app.set_tenant in middleware — RLS depends on it").
- Edge-runtime constraint is real (Node-only `pg` not available in edge middleware), but the mitigation is missing: there is no enforced wrapper. AC3 is not actually demonstrated end-to-end.

**DEVIATION B [LOW] — `require('next/headers')` shim → ACCEPT-with-tightening.**

This is a **P0 production bug** masked by the test fallback (the require throws, so the buggy production path never runs in tests). The shim is silently hiding a real defect.
- Severity reclassified: **MEDIUM (production-blocking) — not LOW**. The shim is a correctness landmine because tests cannot detect prod-only failure.

### Verdict reasoning

REWORK is required because:
1. **AC3 is not actually tested** — the test exercises the SQL primitive, not the AC's "server action" clause. There is no `withOrgContext` wrapper, no Server Action that calls `set_org_context`, and no test asserting that "a query inside a Server Action sees the correct `current_org_id`". This is the highest-severity foundation security control and it is unenforced.
2. **`cookies()` is awaited synchronously** in `supabase-server.ts` against Next 16, which makes `createServerSupabaseClient()` broken in production runtime. Hidden by the test shim's catch-all fallback.
3. **`/auth/` bypass uses `startsWith` without segment boundary** — any future `/auth/<anything>` route silently bypasses idle-timeout enforcement.

Issues 1 and 2 are P0 (security and correctness landmines respectively). Issue 3 is P1 (defense-in-depth gap that becomes critical the moment any non-public route is added under `/auth/`).

### REWORK list (exact)

1. **Add `withOrgContext` Server Action wrapper** in `apps/web/lib/auth/with-org-context.ts`.
