### CRITICAL: middleware.ts ALREADY EXISTS — DO NOT REPLACE
`apps/web/middleware.ts` was created by T-022 (GREEN) and exports `createMiddleware(routing)` from next-intl. T-011 MUST chain Supabase session validation before next-intl, not overwrite it.

Recommended merge pattern (do NOT put session logic in the next-intl middleware):
```ts
// apps/web/middleware.ts  ← T-011 GREEN must produce this merged version
import { createServerClient } from '@supabase/ssr';
import createIntlMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';
import { checkIdleTimeout } from './lib/auth/session-check';

const intlHandler = createIntlMiddleware(routing);

export default async function middleware(req: NextRequest) {
  // 1. Supabase session check (runs first — before i18n routing)
  const sessionResponse = await checkIdleTimeout({
    accessToken: getTokenFromCookie(req),
    path: req.nextUrl.pathname,
    idleTimeoutMin: 60, // read from tenant_idp_config.idle_timeout_min
  });
  if (sessionResponse) return sessionResponse; // 401 or redirect

  // 2. Delegate to next-intl for locale routing
  return intlHandler(req);
}
```
The `checkIdleTimeout` export from `lib/auth/session-check.ts` is what AC2 tests import.

### CRITICAL: Non-spoofable org context — middleware contract
Per T-011.json risk_red_lines and 002-rls-baseline.sql:
1. Middleware reads JWT `sub` (verified by Supabase — non-spoofable)
2. Looks up `users.org_id` FROM DB using the verified sub
3. Registers `app.session_org_contexts (session_token, org_id)` via owner/service-role connection
4. Calls `app.set_org_context(session_token, org_id)` on the app connection
5. NEVER reads org context from request headers or client-supplied params

The `app.set_org_context(uuid, uuid)` function (in 002-rls-baseline.sql) raises `SQLSTATE 28000`
if the `(session_token, org_id)` pair is not registered — this is the RLS red line.

### (auth) route group vs i18n locale prefix
The `app/(auth)/` route group does NOT conflict with `app/[locale]/` because route groups
(parenthetical) are transparent to Next.js routing — they don't add URL segments. However,
verify that `app/(auth)/actions.ts` is a Server Action file, not a page route. If auth pages
(e.g. `/login`) need locale prefixes, they should live at `app/[locale]/(auth)/login/page.tsx`.
