Notes for implementer:
- Migration 013-idempotency.sql must create idempotency_keys table with: transaction_id (UUID PK), org_id (UUID NOT NULL), request_hash (TEXT), response_json (JSONB), created_at (TIMESTAMPTZ DEFAULT now())
- RLS policies must isolate by org_id (app.current_org_id())
- packages/server/src/idempotent.ts must export withIdempotency<T>(transactionId, requestPayload, handler, orgId) that:
  1. Validates transactionId is UUID v7 (version nibble = 7) or throws 'invalid_transaction_id'
  2. Hashes requestPayload deterministically
  3. Attempts INSERT with ON CONFLICT DO NOTHING to prevent race conditions
  4. On hit with matching hash: returns stored response_json
  5. On hit with mismatched hash: throws 'idempotency_conflict'
  6. On miss: runs handler, stores result, returns response
- Avoid re-running side-effects on replay — that is the contract (R14)
- Test pattern uses it.skip when no DATABASE_URL to enable CI integration tests
