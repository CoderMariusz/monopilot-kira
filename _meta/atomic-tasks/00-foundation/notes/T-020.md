Notes for implementer:
- Migration path: `packages/db/migrations/012-manufacturing-ops.sql` (reserved per STATUS.md)
- Seed path: `packages/db/seeds/manufacturing-operations.sql`
- Table DDL per PRD §9.1: `Reference.ManufacturingOperations(id UUID PK, org_id UUID NOT NULL REFERENCES organizations(id), operation_name TEXT NOT NULL, process_suffix TEXT NOT NULL CHECK (process_suffix ~ '^[A-Z0-9]{2,4}$'), description TEXT, operation_seq INT, industry_code TEXT NOT NULL, is_active BOOLEAN DEFAULT TRUE, marker TEXT DEFAULT 'APEX-CONFIG')` + `UNIQUE(org_id, process_suffix)`
- **PRD uses `tenant_id` in the DDL box** (line 541 of PRD), but the implementer note + Wave0 v4.3 red line mandate `org_id` for business scope. Use `org_id` — this is a known PRD inconsistency. The task spec JSON, the mission brief, and the risk_red_lines all say `org_id`.
- ENABLE + FORCE RLS with `org_id = app.current_org_id()` policy + GRANT SELECT, INSERT, UPDATE, DELETE to app_user (matches 011-departments.sql pattern)
- Seed resolves Apex org via `external_id = 'apex'` (same pattern as apex-departments.sql)
- 16 seed rows total: 4 industries × 4 operations
- marker = 'APEX-CONFIG' on every seed row
- ON CONFLICT (org_id, process_suffix) DO NOTHING for idempotency
- Do NOT hardcode operation_name in test code — tests read operation names only from the DB or count rows by industry_code
- The static shape tests are mutation-resistant: they read the actual SQL file text and regex-match specific constraint syntax
- The integration tests use `getOwnerConnection()` for DDL/setup and can use `getAppConnection()` for assertion queries under RLS (AC1-AC3 all go through ownerPool since they test constraint enforcement, not RLS isolation)
- ECONNREFUSED failures in the 3 integration `describe` blocks are the same pre-existing env pattern seen in T-019. The 14 integration tests will properly skip on `runIntegrationSuite = describe.skip` when `DATABASE_URL` is absent.
- **Important vitest quirk**: `describe.skip` suppresses test execution but `beforeAll` still fires, causing ECONNREFUSED to appear even for skipped suites. This is pre-existing behavior (T-019 REVIEW noted it). Non-blocking.
