**Concern 5 — app-role.test.ts `const { Pool } = pg; new Pool()` selector analysis**
PASS (agent's claim is correct). The ESLint selector `NewExpression[callee.object.name='pg'][callee.property.name='Pool']` requires `callee` to be a MemberExpression with `object.name === 'pg'` — i.e., `new pg.Pool(...)`. When Pool is destructured first (`const { Pool } = pg; new Pool(...)`), the AST node for the `new` expression has `callee` as an `Identifier` with `name === 'Pool'`, not a MemberExpression. The selector does NOT match. The agent's statement that no suppression is needed for this file is technically correct.
HOWEVER: this is a logical gap — the drift gate only blocks `new pg.Pool(...)` syntactically but misses the destructuring pattern. This is a pre-existing limitation of T-055's rule; not a T-058 regression. app-role.test.ts was already using this pattern before T-058.
