### Implementation flow (recordMigrationRun)
1. **RBAC guard**: `SELECT 1 FROM user_roles ur JOIN roles r ON ur.role_id = r.id
   WHERE ur.user_id = $actorUserId AND r.org_id = $orgId AND r.slug = 'org.platform.admin'`.
   If no row → write audit_events (action='rbac.denied' OR action='tenant.migration.run',
   retention_class='security', actor_user_id=$actorUserId, request_id=randomUUID()) and
   return `{ success: false, statusCode: 403, error: 'forbidden' }`.
2. **App-layer FK guard** (T-038 carry-forward CARRY-FORWARD-1): `SELECT 1 FROM organizations
   WHERE id = $tenantId`. If absent → throw or return error.
3. **UPSERT** tenant_migrations:
   - `INSERT INTO tenant_migrations (tenant_id, component, current_version, target_version,
     last_run_at, status, failure_reason) VALUES (...)`
   - On status='succeeded': `current_version = $target` (the bump)
   - On status='failed': `current_version = COALESCE(existing.current_version, ...)` —
     do NOT bump
   - `ON CONFLICT (tenant_id, component) DO UPDATE SET ...`
4. **Outbox emit** (T-008 helper):
   - status='succeeded': event_type='tenant.migration.run',
     payload includes `retention_class:'operational'`
   - status='failed': event_type='tenant.migration.run.failed',
     payload includes `action:'tenant.migration.run.failed'` and `failure_reason`
5. Wrap (3) + (4) in a single transaction so outbox + table write are atomic per
   the transactional outbox pattern (T-008).

### Implementation flow (advanceCohort)
1. RBAC guard (same as above; on failure write audit row + return 403).
2. **Single transaction**:
   - `BEGIN`
   - `SELECT tenant_id FROM tenant_migrations WHERE component=$component
       AND cohort=$fromCohort AND status='succeeded'
       AND last_run_at < now() - INTERVAL '15 minutes' FOR UPDATE`
   - For each row: `UPDATE tenant_migrations SET cohort=$toCohort WHERE tenant_id=...
       AND component=...` AND insert outbox row `tenant.cohort.advanced`.
   - `COMMIT`
   - On error → `ROLLBACK` rethrows. The atomicity test (AC3 #2) verifies all-or-nothing.
3. Return `{ success: true, advancedCount: N }`.

### Boundary precision (AC3 #5)
`last_run_at < now() - INTERVAL '15 minutes'` — strict `<` so @15m1s advances and @14m59s
does NOT. Mutation `<= ` → boundary @14m59s advances → fails the boundary test.

### RBAC carry-forward
- T-014 seed trigger creates `org.access.admin` and `org.schema.admin` only. The
  `org.platform.admin` role is **system-scoped per task spec** — the test seeds it
  manually via owner connection (matches T-014 RED-test pattern). Implementer should
  NOT auto-seed it on org creation; it is an operations role granted out-of-band.
- T-014 audit constraint requires `action='role.assigned'` rows to use
  `retention_class='security'`; the AC4 audit row uses
  `action='tenant.migration.run'` (or `rbac.denied`), so the T-014 CHECK does not
  apply. retention_class='security' is enforced by the implementer per task spec.

- **No window collapse below 15 min** — AC3 #3 + #5 pin this.
