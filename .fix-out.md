### What changed

When a WAC debit would drive `item_wac_state` below zero, behavior is now:

1. **Still clamps** qty/value to zero — stock move is **not** rolled back (per arbitration).
2. **Emits `finance.wac.underflow`** into `outbox_events` with:
   - `available_qty_kg` / `available_value` (pre-debit pool)
   - `delta_qty_kg` / `delta_value` (attempted debit)
   - `attempted_post_qty_kg` / `attempted_post_value` (raw post-delta before clamp)
   - optional `source_ref` when caller supplies it
3. **Consume + ship** pass `sourceRef` linking the anomaly to `wo_material_consumption` / `shipment`.

Console-only `console.warn` for clamp cases is removed.
