# Domain Glossary — Wave0 Locked Terms

Source authority: `docs/prd/00-FOUNDATION-PRD.md` §W0-v4.3 (2026-05-03).
This file is the canonical reference for all Foundation domain terms and legacy aliases.
Downstream module tasks (01-NPD, 02-SETTINGS, 03-TECHNICAL, etc.) must not contradict these definitions.

> **LOCK STATUS:** Wave0 locked. No new business semantics may be introduced here.
> All changes require a §W0-vX.Y PRD amendment and a corresponding T-0XX task.

## Key rules summary

1. **`org_id` vs `tenant_id`:** Business tables use `org_id`. `tenant_id` is control-plane only and must NOT appear in new business tables, RLS policies, or module contracts.
2. **`FA` and `fa.*` are legacy aliases only:** They exist for D365/legacy migration compatibility. All new code, events, DDL, and documentation must use `FG`/`fg.*` as the canonical term/prefix.
3. **Canonical flow:** Brief → NPD Project → FG / Product. Not Brief → FA.
4. **D365 is optional:** It is never a source of truth or a gate for factory release.
5. **`factory_spec` and `shared_bom`:** Foundation owns the skeleton; Technical owns implementation and approval.
