## Decisions

1. **Do not greenfield** — valuation route existed; only wired residual gaps.
2. **Do not write design note** — honest foundation slice is shippable without half-building a ledger.
3. **Permission OR-list** — module nav stays on `fin.costs.read`; valuation accepts mig-199 `fin.valuation.view` too so role matrices match PRD §3.
4. **FIFO / mig-199 dormant tables deferred** — `inventory_cost_layers` has no writers; surfacing empty FIFO UI would mislead.

## Ambiguities resolved

- **Which permission?** Both `fin.valuation.view` (199) and `fin.costs.read` (292/nav) — accept either via OR.
- **item_cost_history on Finance?** Stays on Technical (dual-ownership comment in technical/cost/history); Finance consumes WAC state only for this slice.
