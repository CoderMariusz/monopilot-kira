### Changed

- **[T-059] PRD marker discipline sweep — 87 unmarked headings in `docs/prd/00-FOUNDATION-PRD.md`**: Added ADR-035 markers (`[UNIVERSAL]`, `[APEX-CONFIG]`, `[EVOLVING]`, `[LEGACY-D365]`) to all 87 previously-unmarked headings. Classification: 57 × `[UNIVERSAL]` (architecture rules, tech stack, schema/rule-engine/multi-tenant foundations, cross-cutting requirements, success criteria), 7 × `[APEX-CONFIG]` (Apex persona tables, 7-dept taxonomy, role-naming decision, bakery template example, industry seed data), 13 × `[EVOLVING]` (PRD writing phases, build sequence, open-items sections, candidate/deferred ADRs), 1 × `[LEGACY-D365]` (Phase B.2 migration from Process_A/B/C/D v7 placeholders). 10 navigation-only sub-headings (§15 reference lists, §9.1 cross-reference pointers) added to allowlist in `scripts/check-markers.mjs` with justification comments. Also fixed a pre-existing normalization bug in `isAllowedUnmarkedHeading` that prevented `§N — Title` patterns from matching the existing `EXECUTIVE_FRONT_MATTER` allowlist (leading em-dash was not stripped after §-prefix removal). `node scripts/check-markers.mjs docs/prd/00-FOUNDATION-PRD.md` now exits 0.

### Fixed

- **[HOTFIX T-041] Replace 5× `expect(true).toBe(true)` placeholders in sw.test.ts with real source-parse assertions** (`apps/web/app/__tests__/sw.test.ts`): The original test file for the PWA service-worker scaffold contained six tests that were pure stubs — all six passed unconditionally (`expect(true).toBe(true)`) with comments deferring to T-042 E2E. The entire AC family (skipWaiting, clientsClaim, precacheEntries, NetworkFirst strategy, CacheFirst strategy) was unverified by any assertion. Hotfix replaces all 6 placeholders: 5 become genuine source-parse assertions reading `sw.ts` as text (skipWaiting, clientsClaim, __WB_MANIFEST guard, defaultCache import, runtimeCaching wiring) plus 2 parse the installed `@serwist/next/dist/index.worker.mjs` to assert NetworkFirst with `networkTimeoutSeconds: 10` for `/api/` routes and CacheFirst (`next-static-js-assets` cache) for `/_next/static` JS. The offline-fallback AC3 behaviour (browser SW lifecycle) is explicitly moved to `it.skip` with a pointer to T-042 E2E. PRD-vs-code drift documented: AC3 specifies 5s timeout; shipped `defaultCache` uses 10s — the new test pins 10s and flags carry-forward to T-042. Test result: 55 passed, 1 skipped (was 54 passed, 0 skipped with 6 vacuous passes).

- **[HOTFIX T-040] AC3 RLS tests were vacuous (metadata-only) — replaced with genuine cross-org SELECT** (`packages/db/__tests__/r13-business-tables.test.ts`): The original AC3 block (10 tests) only checked `pg_class.relrowsecurity = true` and `pg_policies.length > 0` — never connecting as `app_user` or performing a cross-org SELECT. A permissive `USING (true)` policy would have kept those tests green while providing zero isolation. Hotfix replaces the 10 metadata-only tests with 6 genuine tests (1 sanity + 5 cross-org, one per placeholder table) following the rls.cross-org.integration.test.ts pattern: superuser INSERTs 1 row per org, `app_user` opens a transaction under org A context (asserts count = 1), then org B context (asserts count = 1). The new tests fail if the `USING (org_id = app.current_org_id())` predicate is dropped or weakened. All 29 tests pass; skip cleanly without DATABASE_URL.
- **[HOTFIX T-046] Replace hardcoded `"Reference.DeptColumns"` / `"Reference.FieldTypes"` literals in `packages/schema-runtime/src/compile.ts:39,46` with `RefTables.DeptColumns` / `RefTables.FieldTypes` from `@monopilot/reference`** — first drift caught by the T-046 gate in principle; proves ESLint coverage must be workspace-wide (T-055) to catch `packages/*` drift automatically.
- **[HOTFIX T-019] RLS on `Reference.Departments`** (`packages/db/migrations/011-departments.sql`): Added `ENABLE ROW LEVEL SECURITY`, `FORCE ROW LEVEL SECURITY`, and `Departments_org_isolation` policy (`org_id = app.current_org_id()`) — the only Wave A business table missing RLS, identified by the 2026-05-07 Foundation Wave A consistency audit (finding D). Cross-org isolation verified by two new integration tests in `departments.integration.test.ts`.

### Added

- **[T-127] Authorization Policies screen — SET-011b** (`apps/web/app/(admin)/settings/authorization/page.tsx`): Implements the UX SET-011b two-card layout for the Authorization Policies settings screen. NPD policy card shows policy code, Enabled/Disabled status badge, request-permissions combobox (edit mode), invariant banners ("requires a new released version", "self-authorization is never allowed"), and per-card typed blockers. Technical policy card shows `technical_product_spec_approval_gate_v1`, a read-only factory-use-blocking-locked-on banner, and enforces `factoryUseBlockingLocked: true` via `enforcePolicyInvariants` (immutable, cannot be disabled). Read-only mode (no `settings.authorization.edit` permission) renders a "Read-only" notice, both cards visible, and suppresses Save/Discard controls. Save path validates audit-reason presence, enforces invariants, and forwards to T-126 `onSave` prop with the audit reason; Save is disabled when any policy has typed blockers. RTL test suite (5/5 pass) serves as UI-PROTOTYPE-PARITY-POLICY.md fallback evidence (Playwright browser runtime not configured for this worktree).

- **[T-128] Schema Shadow Preview screen — SET-034** (`apps/web/app/(admin)/settings/schema/preview/page.tsx`): Implements the UX SET-034 split-layout preview screen (40 % left / 60 % right). Left panel: draft-column combobox populated from department draft columns. Right panel: rendered sample form showing generated runtime schema and sample data (read-only inputs) with a preview-only notice (role=alert). "Publish this Column" button delegates to the existing SET-031/T-097 Step 8 `publishDeptColumnDraft` path with concurrent-edit handling (`CONCURRENT_SCHEMA_VERSION` surfaces a version-conflict alert). All preview operations are non-mutating — `upsertDeptColumnDraft` is never called. No-draft, loading, permission-denied, and schema-generation error states render informational UI only, no publish button. RTL test suite (8/8 pass) serves as UI-PROTOTYPE-PARITY-POLICY.md fallback evidence (Playwright not configured for this worktree; no canonical SET-034 prototype exists — spec-driven against `docs/prd/02-SETTINGS-PRD.md §6.4/§6.6` and `SET-034` UX doc).

- **Baseline DB schema migration** (`packages/db/migrations/001-baseline.sql`, `packages/db/schema/baseline.ts`): Establishes the three-table control-plane/business-scope foundation. `tenants` is the control-plane root (id UUID PK, name, region_cluster CHECK `eu|us`, data_plane_url, created_at). `organizations` is the canonical business scope root (id UUID PK used as `org_id` everywhere, tenant_id nullable FK to tenants, name, industry_code CHECK `bakery|pharma|fmcg|generic`, external_id, plus full R13 identity columns: created_at, created_by_user, created_by_device, app_version, model_prediction_id, epcis_event_id, schema_version). `users` is org-scoped (id UUID PK, org_id UUID NOT NULL FK to organizations, email CITEXT unique per org, display_name, external_id, full R13 columns). Business-facing indexes use `org_id`; `tenant_id` is not propagated to `users`. No RLS policies (owned by T-007). Integration tests verify R13 column presence and DB-level industry_code rejection via information_schema (T-006 / TASK-000210).
- **[T-001] Settings permission enum extension** (`packages/rbac/src/permissions.enum.ts`): Appended 8 settings-domain permission strings to the `Permission` const-object (`settings.org.read`, `settings.org.update`, `settings.users.create`, `settings.users.deactivate`, `settings.users.invite`, `settings.roles.assign`, `settings.audit.read`, `settings.impersonate.tenant`) per PRD §3 (SETTINGS-PRD). Added `ALL_SETTINGS_CORE_PERMISSIONS` as a typed `readonly Permission[]` export for safe iteration. Vitest suite extended to cover regex format, uniqueness, exact contents, typed export, CODEOWNERS guard, canonical-set membership, SOD pairs, and legacy alias normalization (8 tests pass). `packages/rbac/src/permissions.enum.ts` gated behind `@monopilot/architect` review via CODEOWNERS (TASK-003246).
- **RBAC permission enum** (`packages/rbac/src/permissions.enum.ts`): Source-of-truth lock for org-scoped permission strings. Exports `Permission` const-object with canonical values (`org.access.admin`, `org.schema.admin`, `org.scim.write`, `fg.create`, `fg.edit`, `brief.convert_to_npd_project`, `ref.edit`, `audit.read`, `outbox.admin`, `impersonate.org`), `LegacyPermissionAlias` map for backward-compat aliases (`fa.create → fg.create`, `fa.edit → fg.edit`, `brief.convert_to_fa → brief.convert_to_npd_project`), `ALL_PERMISSIONS` (canonical values only, no duplicates), `SOD_EXCLUSIVE_PAIRS` locking the Org Admin / Schema Admin separation-of-duties constraint, and `normalizePermission()` helper that resolves legacy aliases and throws on unknown strings.
- **RBAC permission tests** (`packages/rbac/src/__tests__/permissions.test.ts`): 14 passing tests covering format validation, SoD pair integrity, fg.* canonical contract, legacy alias normalization, and CODEOWNERS lock.
- **CODEOWNERS entry**: `permissions.enum.ts` gated behind `@monopilot/architect` review to prevent accidental drift (T-004 / TASK-000208).
- **RLS baseline migration** (`packages/db/migrations/002-rls-baseline.sql`): Enables row-level security on `public.organizations` and `public.users` using a non-spoofable org-context pattern. Introduces `app` schema with private `session_org_contexts` / `active_org_contexts` tables, a `SECURITY DEFINER` `app.set_org_context(session_token, org)` function that validates tokens written by trusted middleware, and `app.current_org_id()` used by all policies. `app_user` has all context-table privileges revoked and cannot spoof `app.current_org_id` via a direct `SET` GUC. `public.tenants` remains control-plane-only with `app_user` access revoked (T-007 / TASK-000211).
- **RLS cross-org integration tests** (`packages/db/__tests__/rls.cross-org.integration.test.ts`): 14 passing tests connecting as `app_user` (asserted non-superuser), covering: org-scoped SELECT isolation, cross-org INSERT rejection, direct GUC spoof attempt that proves `app.current_org_id()` is unaffected, `pg_policies` assertions confirming `app.current_org_id()` is used and `current_setting('app.tenant_id')` is absent (T-007 / TASK-000211).
- **Marker discipline ADR** (`_foundation/decisions/ADR-035-marker-discipline.md`): Codifies the four marker tags ([UNIVERSAL], [APEX-CONFIG], [EVOLVING], [LEGACY-D365]) and the Wave0 domain naming rule — business-domain docs and tasks use `org_id` and `fg.*` prefix; `fa.*`/`FA` are permitted only as [LEGACY-D365] compatibility aliases. Conditionally cross-references `_foundation/glossary/domain-terms.md` without requiring T-048 to have run (T-005 / TASK-000209).
- **15-module registry seed** (`_foundation/registry/modules.json`): 16-entry JSON registry (ids 00–15: 00-FOUNDATION plus 15 business modules) sourced from PRD §4.3, with id, slug, phase, build_order, file, dependencies, and `business_scope_column: "org_id"` on every entry (T-005 / TASK-000209).
- **Marker discipline checker** (`scripts/check-markers.mjs`): Node.js script that validates PRD/ADR headings carry a recognised marker tag; exits non-zero for any unmarked heading and zero for conformant files. Includes an allowlist for executive front matter. Confirmed: exits 0 for ADR-035 and exits 1 for an unmarked fixture (T-005 / TASK-000209).
