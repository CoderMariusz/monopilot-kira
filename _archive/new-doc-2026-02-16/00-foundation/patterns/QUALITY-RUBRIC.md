# Quality Rubric

## Overview
Standards for evaluating work quality across agents.

## Code Quality

### Excellent (5)
- Clean, readable code
- Comprehensive tests
- Full documentation
- No code smells
- Follows all conventions

### Good (4)
- Clear code
- Good test coverage
- Adequate documentation
- Minor issues only

### Acceptable (3)
- Working code
- Basic tests
- Minimal documentation
- Some cleanup needed

### Needs Work (2)
- Functional but messy
- Limited tests
- Poor documentation
- Multiple issues

### Unacceptable (1)
- Hard to understand
- No tests
- No documentation
- Fails standards

## Documentation Quality

### Excellent (5)
- Complete and accurate
- Well-organized
- Clear examples
- Kept up to date

### Good (4)
- Covers key areas
- Understandable
- Some examples

### Acceptable (3)
- Basic coverage
- Usable
- May need updates

### Needs Work (2)
- Incomplete
- Unclear sections
- Outdated

### Unacceptable (1)
- Missing or wrong
- Confusing
- Misleading

## Process Quality

### Excellent (5)
- All steps followed
- Great communication
- Smooth handoffs
- Proactive updates

### Good (4)
- Process followed
- Good communication
- Clean handoffs

### Acceptable (3)
- Process mostly followed
- Adequate communication
- Handoffs work

### Needs Work (2)
- Process gaps
- Communication issues
- Handoff problems

### Unacceptable (1)
- Process ignored
- Poor communication
- Broken handoffs

## Applying the Rubric
1. Evaluate against criteria
2. Score each dimension
3. Document reasoning
4. Identify improvements
5. Track over time
